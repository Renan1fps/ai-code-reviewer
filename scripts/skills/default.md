# Revisão Geral

Faça uma revisão completa do código com base nas boas práticas gerais de desenvolvimento.

## O que avaliar

- Clareza e legibilidade do código
- Nomenclatura de variáveis, funções e classes
- Complexidade desnecessária ou funções muito longas
- Tratamento de erros em operações assíncronas
- Código duplicado (violações do princípio DRY)
- `console.log` ou código de debug em produção
- Comentários desatualizados ou código comentado
- Possíveis bugs ou comportamentos inesperados

## Tom da revisão

Seja construtivo e objetivo. Aponte o problema, explique o porquê e sugira como corrigir.
